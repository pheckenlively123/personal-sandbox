<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host script → running sandbox | `openshell sandbox exec` drives the round-trip curl into the isolated sandbox; the inference path is the asset under test. |
| sandbox → inference.local gateway | The gateway strips the placeholder credential and injects the real host-side subscription credential; the placeholder must never be a real secret. |
| operator → host credential store | `--from-existing` reads `~/.claude/.credentials.json` host-side; documented as a deliberate human step, never automated. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Spoofing/false-pass | `run_inference_round_trip` (D-06) | mitigate | Detect success by parsing `.content | length > 0` in the JSON body, NOT curl exit code (which is 0 even on a gateway error body — Pitfall 2). A false PASS could mask a broken inference path Phase 4 depends on. |
| T-03-08 | Information Disclosure | round-trip request header | mitigate | The request sends `x-api-key: placeholder` — never a real credential. The gateway injects the real subscription token host-side; the sandbox never holds it (NET-03). |
| T-03-09 | Information Disclosure | README credential guidance | mitigate | README documents `--from-existing` (reads host OAuth store) and explicitly forbids baking credentials into the image or Dockerfile; no secret values appear in README. |
| T-03-10 | Tampering | `/v1` double-path / wrong base URL | mitigate | README and the round-trip use `inference.local` without a trailing `/v1` base (single `/v1/messages` only on the raw curl); checks assert `! grep inference.local/v1` in README and `! grep inference.local/v1/v1` in rebuild.sh. |
| T-03-11 | Availability | round-trip blocking the rebuild | accept | The round-trip is intentionally non-fatal (D-06): it WARNs and `return 0`s so an unconfigured-provider state never blocks handing a correctly-isolated sandbox to the operator. The blocking guarantees live in 03-01 (NET-04/NET-05). |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this phase (RESEARCH.md Package Legitimacy Audit: "Skip"). No install tasks; gate not triggered. |
</threat_model>
