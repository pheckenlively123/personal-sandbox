<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → sandbox image build | Dockerfile pulls + builds tooling at build time (has network); runtime sandbox does not |
| sandbox process → /opt (toolkit) | Landlock now grants read to operator-controlled fork code that `claude --dangerously-skip-permissions` will load and execute |
| sandbox claude binary → 3-host allowlist | Only api.anthropic.com / platform.claude.com / claude.ai reachable; everything else denied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | granting /opt read to a process running `--dangerously-skip-permissions` | accept | `/opt` is read-only (not read_write); the toolkit is an operator-maintained fork cloned at build time (IMG-05, no cooldown — trusted); no untrusted code path writes to /opt at runtime. Accepted trade-off required for RUN-02. |
| T-04-02 | Tampering | toolkit code at /opt mutated at runtime | mitigate | Add /opt to `read_only` only — never `read_write`. Landlock blocks writes. |
| T-04-03 | Information Disclosure | govulncheck binary at world-readable /usr/local/bin | accept | govulncheck is a public Go binary; no secret embedded; required for the vuln-reviewer audit invocation. |
| T-04-SC | Tampering | govulncheck `go install` install | mitigate | Pin retained via `@${GOVULNCHECK_VERSION}` (PIN-03); resolved host-side under cooldown by build-and-lock.sh; PIN-07 verifier (scripts/verify-pins.sh) already fails the build on a post-cooldown publish date. No new npm/pip/cargo installs in this plan. |
</threat_model>
