<objective>
Deliver `rebuild.sh` — the single, idempotent, top-level orchestrator that ties the
whole Phase 2 spine together end-to-end: preflight tools, resolve+build via the Phase 1
seam (now with the `--build-date` flag from Plan 02-01), tag `:latest`, tear down any

`--audit` egress-review subcommand and operator README docs.

runs `./rebuild.sh` and ends up with a Ready sandbox whose `/claudeshared` accepts a
write that lands host-owned — the full working capability, not a layer.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → rebuild.sh (host privileges) | Script runs podman + openshell with the operator's host privileges; touches $HOME/claudeshared bind mount and tears down images |
| openshell/podman CLI output → rebuild.sh control flow | Teardown decisions are made from CLI stderr strings (untrusted text crosses here) |
| $HOME → --driver-config-json (JSON) | The expanded $HOME path is interpolated into a JSON string handed to openshell |
| teardown (podman rmi/prune) → host image store | Destructive operations whose blast radius must be bounded to this project's images |
| in-sandbox agent → /claudeshared (Landlock) | The sandbox agent (Landlock-confined) writes to the host bind mount; policy.yaml is the boundary control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering/Injection | $HOME → `--driver-config-json` JSON | mitigate | Validate `CLAUDESHARED_ABS` is an absolute path with no quote/backslash/JSON metacharacters before interpolating; expand $HOME (never `~`) — rebuild.sh Step 4 (Task 2). |
| T-02-05 | Tampering | openshell CLI output → teardown branch | mitigate | Never `eval` CLI output; tolerate-absent uses exact `grep -q "sandbox not found"` string match; any other non-zero delete is a hard error (Pattern 1). |
| T-02-06 | Denial of Service / destructive blast radius | `podman rmi` / `podman image prune` in teardown | mitigate | rmi targets ONLY `localhost/claude-sandbox:*` (enumerated via `--filter reference=`); `prune --force` removes only dangling layers. Never `rmi -a` or untargeted prune of named images — bounds blast radius to this project (RESEARCH open question #3). |
| T-02-07 | Elevation of privilege / policy weakening | policy.yaml | mitigate | policy.yaml grants ONLY `/claudeshared` read_write and adds NO network section; zero-egress enforcement is untouched and deferred to Phase 3 (D-07 scope boundary, Task 1). |
| T-02-08 | Information disclosure | `--audit` (`openshell logs`) | accept | Audit surfaces logs to the operator's own terminal only; it is read-only, never weakens policy, and is the intended BLD-05 review surface (D-07). |
| T-02-09 | Spoofing / wrong image | `openshell sandbox create --from` | mitigate | Always pass the full date-pinned local ref `localhost/claude-sandbox:<date>`; never `--from .` (Docker-daemon path) or `--from-existing` (CLAUDE.md anti-patterns, Pitfall 3). |
| T-02-SC | Tampering | npm/pip/cargo installs | n/a | No package installs in this phase (RESEARCH "No New Packages"); supply-chain install gating was handled in Phase 1. No legitimacy checkpoint required here. |
</threat_model>
