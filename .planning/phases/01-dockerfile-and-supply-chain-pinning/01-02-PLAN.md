<objective>
Complete the resolve/build/lock/verify loop with its safety gate: a host-side pin-held verifier
(PIN-07) that re-queries every recorded package's publish date and fails the pipeline closed if
any pin postdates the cooldown date, wired in as the final step of `build-and-lock.sh`. Add the
two negative/assertion tests that prove the loop's guarantees: the dnf cache actually busts when
the cooldown date rolls (Criterion #2), and a violating pin makes the pipeline exit non-zero
(Criterion #5).

Purpose: PIN-07 is the primary mitigation for the whole phase's supply-chain threat surface —
it must fail closed (a verifier that fails open defeats the guarantee). This plan refines the
Plan 01 skeleton into the full, self-verifying loop.
Output: scripts/verify-pins.sh, the verifier wiring in build-and-lock.sh, and two test scripts.
</objective>

<artifacts_produced>
## Artifacts this plan produces

- `scripts/verify-pins.sh` — host PIN-07 verifier. Reads `versions.lock` + `versions-npm.json`,
  re-queries proxy.golang.org / registry.npmjs.org for each package's true publish date,
  compares to `${cooldown_date}T23:59:59Z`, exits 1 on the first violation, exits 0 if all clean.
  Fails closed: any missing input, malformed lock, or registry-query failure also exits non-zero.
- `scripts/build-and-lock.sh` (modified) — appends `verify-pins.sh` as the terminal step so the
  overall pipeline exit code reflects PIN-07.
- `tests/test-cache-bust.sh` — builds twice with differing COOLDOWN_DATE, asserts the dnf layer
  is not `CACHED` on the second build (Criterion #2 / IMG-02).
- `tests/test-pin-held.sh` — seeds a post-cooldown publish date into a copy of versions.lock and
  asserts `verify-pins.sh` exits non-zero (Criterion #5 negative-path proof).
</artifacts_produced>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| versions.lock / versions-npm.json -> verifier | build-produced artifacts are the verifier's input; a tampered or incomplete lock must not pass |
| verifier -> public registries | re-queried publish timestamps are untrusted external input |
| verifier itself -> pipeline exit code | a verifier that fails OPEN would silently defeat PIN-07 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Elevation of Privilege | verifier fails OPEN (exits 0 on missing input / registry error), defeating PIN-07 | mitigate | `set -euo pipefail` + explicit fail-closed: missing versions.lock, malformed JSON, or registry-query failure all exit non-zero; tests/test-pin-held.sh proves a violation is caught. |
| T-01-08 | Tampering | transitive npm dep postdates cooldown but top-level pins look clean | mitigate | Verifier iterates EVERY transitive pair from versions-npm.json and re-queries each (D-04), not just the three top-level pins. |
| T-01-09 | Spoofing | stale dnf cache silently serves pre-roll RPMs | mitigate | tests/test-cache-bust.sh asserts no `CACHED` on the dnf layer across differing COOLDOWN_DATE (IMG-02 / Criterion #2). |
| T-01-SC | Tampering | npm package installs (supply chain) | mitigate | This plan's verifier IS the fail-closed PIN-07 gate that backstops the install steps; carried forward from Plan 01. |
</threat_model>
