# Quick Task 260630-jm8 — Plan

## Objective

Add a fail-closed, RUN-05-style preflight that asserts the OpenShell gateway's
`supervisor_image` is pinned to a non-floating tag **before** `openshell sandbox
create` runs — closing the supply-chain gap that let the floating
`ghcr.io/nvidia/openshell/supervisor:latest` tag drift.

## Background (root cause this defends against)

The OpenShell gateway mounts the in-sandbox supervisor binary from a container
image. Its default reference is `ghcr.io/nvidia/openshell/supervisor:latest`
ensured with pull policy `newer`, so the gateway re-pulls `:latest` on every
create. On 2026-06-30 upstream published a new `:latest` build that is newer than
the host's pinned gateway (`openshell 0.0.62`). The version skew makes the
supervisor's in-container network-namespace setup fail with:

```
WARN  ...netns: Failed to delete network namespace
Error:  × Invalid argument (os error 22)
```

The container exits 1, so the gateway can never SSH in and `sandbox create`
reports `"sandbox is not ready"` / `ssh exited with status 255`. The host fix
(already applied) pins the supervisor in `~/.config/openshell/gateway.toml`:

```toml
[openshell.drivers.podman]
supervisor_image = "ghcr.io/nvidia/openshell/supervisor:0.0.62"
```

The repo pins the CLI/gateway to 0.0.62 (brew formula) but never pinned the
supervisor image — that is the gap this preflight makes explicit and enforced.

## Operator decision (locked)

- **Verify + fail closed only.** Read-only. The preflight READS gateway.toml and
  exits 1 with remediation if `supervisor_image` is absent OR its tag is
  `:latest`. It never writes/creates/modifies host config and never restarts the
  gateway. (Mirrors the RUN-05 posture; auto-fix/auto-restart rejected.)
- **Do NOT hardcode the version.** The check requires a *pinned, non-floating*
  tag — any tag other than `latest` passes — so it survives a future
  `brew upgrade openshell` + re-pin without a code change.

## Tasks

### Task 1 — `scripts/preflight-supervisor-pin.sh` (new)
Model on `scripts/preflight-gateway-bind-mount.sh`:
- `#!/usr/bin/env bash`, `set -euo pipefail`, `BASH_SOURCE` path resolution,
  diagnostics to stderr.
- XDG-aware config path: `${XDG_CONFIG_HOME:-$HOME/.config}/openshell/gateway.toml`;
  optional positional arg overrides (for the guard gauntlet).
- Section-aware **awk** parse (no python/eval of config): track `[section]`
  headers, strip `#` comments, and read the `supervisor_image` value **only**
  within `[openshell.drivers.podman]`. Emit the resolved value, not a boolean,
  so the tag can be inspected.
- Fail-closed exits (each with the shared remediation block on stderr):
  - file absent → exit 1
  - awk parse failure → exit 1
  - `supervisor_image` absent under the table → exit 1
  - tag is `latest` (`...supervisor:latest`, or bare repo with no tag which
    podman treats as `:latest`) → exit 1
- PASS (exit 0) when a non-`latest` tag is present; log the resolved pin.
