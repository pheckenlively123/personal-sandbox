## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator shell → rebuild.sh verb | verb takes no user-supplied arguments; pure dispatch |
| rebuild.sh → openshell sandbox exec | bridges host to in-sandbox claude binary running --dangerously-skip-permissions |
| claude (autonomous) → 3-host allowlist + ~/.claude token | autonomous claude can read the subscription OAuth token at ~/.claude/.credentials.json; egress restricted to 3 hosts |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

The OAuth token lives only inside the sandbox. This phase consumes the locked model, does not change it. |
| T-04-05 | Information Disclosure | subscription OAuth token at ~/.claude/.credentials.json reachable by autonomous claude | accept | Egress is binary-scoped to claude and limited to 3 Claude auth/API hosts (TLS passthrough — proxy never decrypts); a compromised plugin cannot exfiltrate the token to an arbitrary host (any non-allowlisted connection gets a 403). Documented accepted trade-off in CLAUDE.md. |
| T-04-06 | Tampering | verb argument injection | mitigate | The `claude` verb accepts no user arguments; `case` dispatch only; no `eval`; flags are literals (V5). |
