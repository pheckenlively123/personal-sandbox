<objective>
Close the single BLOCKER and the cheap WARNING hardening from Phase 01 verification.

The PIN-07 fail-closed guarantee is structurally broken at the cutoff boundary: both
verify-pins.sh and resolve-versions.sh compare ISO-8601 timestamps against the
second-precision cutoff `${COOLDOWN_DATE}T23:59:59Z` using bash/jq lexicographic string
comparison. Because ASCII `.` (0x2E) sorts before `Z` (0x5A), any millisecond-precision
timestamp in the window `23:59:59.000Z`–`23:59:59.999Z` on the cutoff day compares as LESS
THAN the cutoff string, so a violating package published in that window passes the verifier
silently (CR-01). This plan replaces the lexicographic comparison with an EXCLUSIVE
next-day-midnight bound (`CUTOFF_EXCL`) computed via python3, applied consistently in the
verifier, the resolver's Go-proxy loop, and the resolver's jq npm selects — preserving the
INCLUSIVE whole-cutoff-day semantics (Pitfall 2), and adds a boundary-window regression test.

It also folds in the cheap fail-closed/security hardening flagged WARNING in review:
CR-02 (replace eval of registry-controlled resolver output with an allowlist-validated
parse), WR-01 (guard the Dockerfile npm-ls snapshot so JSON is always captured), and WR-02
(surface missing/invalid transitive nodes as violations instead of dropping them). IN-02
(pre-release filtering) is folded into the resolver's Go-proxy selection if it fits cleanly;
otherwise it is recorded in Notes, not silently dropped.

Purpose: a verifier that fails OPEN at the boundary defeats the entire phase's supply-chain
guarantee. This restores PIN-07 to a genuine fail-closed gate.
Output: corrected verify-pins.sh + resolve-versions.sh, an eval-free build-and-lock.sh, a
guarded Dockerfile snapshot step, and a boundary regression case in test-pin-held.sh.
</objective>

  selects use `CUTOFF_EXCL` (strictly-before-next-day-midnight) instead of lexicographic
  `<` / `<=` against `T23:59:59Z`. Optionally restricts Go tags to release form `^v[0-9]+\.[0-9]+\.[0-9]+$` (IN-02).
