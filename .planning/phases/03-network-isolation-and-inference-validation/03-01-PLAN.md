<objective>
Add the three blocking egress-isolation gates to `rebuild.sh` so a rebuild can never hand the operator a sandbox whose zero-egress guarantee is unverified.

This plan delivers, as one vertical slice, the proof chain that makes the sandbox's isolation provable rather than assumed:
1. A **provider preflight** (Step 0, D-03/NET-03) that fails closed before the slow `sandbox create` if no inference provider is registered — mitigating the OpenShell #759 ~290s hang.
2. A **live policy assertion** (Step 5, D-02/NET-04) that queries the created sandbox and aborts if any direct Anthropic endpoint exists.
3. A **blocking egress smoke test** (Step 6, D-05/NET-05) that runs curl from inside the running sandbox and aborts if an outbound request unexpectedly succeeds — directly proving NET-01 (deny-all egress).

Purpose: NET-01 deny-all is already enforced by the OpenShell proxy default (empty `network_policies`). This phase does not *build* the enforcement — it *proves* it on every rebuild and fails closed when proof is missing.
Output: `rebuild.sh` extended with three new functions and their call sites; no new files.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host script → openshell CLI | rebuild.sh consumes CLI stdout/stderr; output is untrusted text that must not be `eval`'d. |
| host script → running sandbox | `openshell sandbox exec` injects curl into the isolated sandbox; the sandbox's network egress is the asset being protected. |
| sandbox → open internet | The OpenShell proxy (10.200.0.1:3128) is the enforcement point; deny-all is the guarantee under test. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing/Tampering | `check_inference_provider` preflight | mitigate | Detect "Not configured" via ANSI-stripped output grep, never the exit code (which is 0 in both states — a false-pass risk). Fail closed with `exit 1`. |
| T-03-02 | Elevation of Privilege | `run_egress_smoke_test` (NET-05) | mitigate | Test two independent targets (api.anthropic.com + example.com); a deny-all guarantee requires BOTH to fail. A single Anthropic-only probe could miss a generic egress hole. PASS condition is curl exit != 0; any exit 0 hard-aborts. |
| T-03-03 | Tampering | `assert_no_anthropic_egress` (NET-04) | mitigate | Query the LIVE sandbox policy (`openshell policy get`), not the static policy.yaml source, so built-in defaults or provider attachments that widen egress are caught. Inverted `jq -e` is fail-closed: a found Anthropic endpoint aborts. |
| T-03-04 | Tampering (supply/config) | egress policy widening | mitigate | rebuild.sh NEVER calls `openshell policy update --add-endpoint` (CLAUDE.md anti-pattern); policy.yaml stays filesystem-only (D-01). The NET-04 assertion is the runtime detector if widening ever occurs. |
| T-03-05 | Information Disclosure | CLI output in logs | accept | rebuild.sh logs `openshell inference get` and `openshell policy get` output, which contain no credential values (credentials live host-side in the gateway provider record, never surfaced). |
| T-03-06 | Tampering | ANSI/control-char injection via CLI output | mitigate | Strip ANSI with `sed 's/\x1B\[[0-9;]*[mK]//g'` before grepping; CLI output is never `eval`'d. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this phase (RESEARCH.md Package Legitimacy Audit: "Skip — no new external packages"). No install tasks; gate not triggered. |
</threat_model>
