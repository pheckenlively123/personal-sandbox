<objective>
Deliver the walking-skeleton happy path of the supply-chain pipeline: a host-side bash
resolver computes the rolling cooldown date and the three pinned versions, a `FROM fedora:44`
Dockerfile installs the complete toolchain (Go RPM, golangci-lint RPM, govulncheck via
`go install`, gsd-core + Claude Code via npm `--before`, toolkit via git clone) at those
pinned versions, and a thin driver script ties resolve -> build -> lock together so
`versions.lock` is produced end-to-end on every run.

Purpose: Prove the resolve -> build -> lock segment of the resolve/build/lock/verify loop
works as one connected vertical slice (NOT three disconnected horizontal layers). This is the
walking skeleton for the whole project — the architectural seam Phase 2's `rebuild.sh` wraps.
Output: Dockerfile, scripts/resolve-versions.sh, scripts/build-and-lock.sh, .dockerignore, and
a generated versions.lock artifact.
</objective>

<artifacts_produced>
## Artifacts this plan produces

- `scripts/resolve-versions.sh` — host resolver. CLI: `--cooldown-days N` (default 4). Stdout:
  sourceable `KEY=VALUE` lines: `COOLDOWN_DATE=YYYY-MM-DD`, `GOVULNCHECK_VERSION=vX.Y.Z`,
  `GSD_CORE_VERSION=X.Y.Z`, `CLAUDE_CODE_VERSION=X.Y.Z`. Exit non-zero on registry failure.
- `Dockerfile` — `FROM fedora:44`. Build ARGs: `COOLDOWN_DATE`, `GOVULNCHECK_VERSION`,
  `GSD_CORE_VERSION`, `CLAUDE_CODE_VERSION`. Emits in-image `/versions-npm.json`,
  `/versions-govulncheck.txt`.
- `scripts/build-and-lock.sh` — driver. Runs resolver, calls `podman build` with build-args,
  extracts snapshots via `podman create`/`cp`/`rm`, writes `versions.lock` (JSON).
- `versions.lock` (generated) — JSON: `cooldown_date`, `build_date`, `cooldown_days`,
  `packages.{govulncheck,@opengsd/gsd-core,@anthropic-ai/claude-code}.{version,publish_date,registry}`,
  `npm_transitive_snapshot` path. Consumed by Plan 02's verifier.
- `versions-npm.json`, `versions-govulncheck.txt` (generated, extracted from image).
- `.dockerignore` — excludes `.planning/`, `.git/`, `*.lock` from build context.
</artifacts_produced>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host resolver -> public registries | proxy.golang.org / registry.npmjs.org responses are untrusted external input parsed by jq |
| host -> podman build (in-image network) | git clone of operator fork + npm/dnf fetches cross into the image at build time |
| `--cooldown-days N` CLI arg -> resolver | operator-supplied integer crosses into date arithmetic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `npm install -g` pin bypass (`@latest` or missing `--before`) silently pulling post-cooldown code | mitigate | Dockerfile uses `@${VERSION}` from ARG + `--before=${COOLDOWN_DATE}` on every npm install; Task 2 verify greps Dockerfile to assert `@latest` absence; Plan 02 host verifier is the fail-closed backstop. |
| T-01-02 | Spoofing | stale cached dnf layer serving outdated RPMs | mitigate | `ARG COOLDOWN_DATE` placed before `RUN dnf update` busts cache on date roll (D-07); proven by Plan 02 cache-bust assertion. |
| T-01-03 | Tampering | unvalidated `--cooldown-days` shell input | mitigate | Resolver validates N is a positive integer, exits non-zero on malformed input (RESEARCH V5). |
| T-01-04 | Denial of Service | registry unavailable during resolve | mitigate | Resolver fails fast with clear stderr + non-zero exit; build does not proceed without resolved versions. |
| T-01-05 | Tampering | toolkit `git clone` pulls untrusted HEAD (IMG-05) | accept | Operator explicitly owns/trusts the fork `pheckenlWork/claude-engineering-toolkit` per CLAUDE.md "Out of Scope: Cooldown pinning for toolkit"; residual risk documented — Phase 4 audits MCP network calls. |
| T-01-06 | Information Disclosure | credential baked into image ENV/ARG | mitigate | No credentials in Dockerfile; CLAUDE.md prohibits it; inference credential is Phase 3 runtime injection only. |
| T-01-SC | Tampering | npm package installs (supply chain) | mitigate | gsd-core + claude-code pre-approved in RESEARCH Package Legitimacy Audit (locked CLAUDE.md decisions, not discovered); `--before` date filter + top-level pin; PIN-07 host gate (Plan 02) fails closed. |
</threat_model>
